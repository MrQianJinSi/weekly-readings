# 一周见闻（2019年第7期）

## 文章

- [Understanding IPFS in Depth(1/5): A Beginner to Advanced Guide](https://hackernoon.com/understanding-ipfs-in-depth-1-5-a-beginner-to-advanced-guide-e937675a8c8a)

  IPFS是一个分布式数据传输协议，我感觉跟之前的P2P传输协议差不多，正好借着这几篇文章了解一下。

- [On Simplicity](https://mxb.at/blog/on-simplicity/)

  作为开发者，都有追逐新潮技术的心里动机。然而技术总是在不停的进化，最新的技术不一定是最合适的技术。这篇文章提出了Simple as a Feature的理念，作者提倡保持项目尽可能的简单。

- [Humans, not Users](https://johannesippen.com/2019/humans-not-users/)

  我非常喜欢这篇文章，现在的产品都流行数据驱动，所有的产品因素都要做A/B Test，来尽可能的提高产品的活跃度、留存率等指标。这种把用户当数字看，而不把用户当人看的态度甚至还受到了吹捧，想想还挺讽刺的。这篇文章提倡的观点“Designing for the whole human, not the user”，只不过是回到了设计的初心：为人类而设计。

- [The Great Divide](https://css-tricks.com/the-great-divide/)

  早年的前端开发包含了HTML/CSS/Javascript这三项内容，而现在我们说前端开发时，一般都特制Javascript，HTML/CSS几乎被视为Javascript的附庸，至于交互设计、可访问性这些内容，更是几乎被前端开发忽视了。这篇文章详细的讨论了这种现象，列举了大家对这种现状的不满，这种现象是暂时的吗，没有人知道。

- [https://kushaldas.in/posts/tracking-my-phone-s-silent-connections.html](https://kushaldas.in/posts/tracking-my-phone-s-silent-connections.html)

  现在人人生活都离不开手机，可否曾想过看看手机每天都连接了什么服务器，都发送了什么数据。Kushal Das监测了自己手机一周的流量，文中是他的手机数据分析结果。

- [How to write a Python web framework. Part I.](http://rahmonov.me/posts/write-python-framework-part-one/)

  网络框架大家肯定都用过，有没有尝试自己写一个呢。其实没有那么复杂啦，这篇文章是一个不错的开始。

## 工具

- [W3C中文排版规范](https://github.com/w3c/clreq)

  我读某些网页，经常会感觉排版不好，但是具体要说那里不好，我也说不上来。这是因为没有一套系统的理论来指导我，这篇W3C中文排版需求其中有大量的术语和规范，我从中学到了不少知识。

- [科幻风格的UI组件库](https://github.com/arwesjs/arwes)

  最近《流浪地球》很火，电影里面有超多的UI界面。如果想尝试做出那样的效果，可以试试这个UI组件库，效果很炫。

## 图书

- [寄生之魔](https://book.douban.com/subject/27081850/)

  看完《流浪地球》之后，产生了重读一遍年轻时看过的科幻小说的冲动，就从罗隆翔开始吧。为啥是罗隆翔呢，因为我隐约记得初中时看的《寄生之魔》是我看过的第一篇科幻小说，然而《寄生之魔》是2003年发表的，那时我已经读高中了，看来我又记忆错乱啦。
